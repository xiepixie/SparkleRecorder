## Document Roles

Semantic recording S2 current note: `semantic-recording debug-smoke` now has an optional `--synthetic-redaction` / `--synthetic-redaction-reason <redacting-reason>` rehearsal path that records explicitly synthetic suppression metadata in JSON/plain output and the debug-smoke evidence sidecar. Finished smoke payloads reload the just-written bundle from disk with `RecordingBundleStore.loadBundleTolerant(from:)` before readiness evaluation, then report `commandPlan` handoff commands, app-facing `SemanticRecordingPreflightPresentation`, persisted counts, structured `persistedBundleCountCheck` memory-vs-persisted match/mismatch details, loaded/missing/failed sidecar diagnostics, `SemanticRecordingBundleReadiness` policy, status, issue counts, issue details and follow-up actions in JSON/plain/sidecar evidence, so authorized live evidence review can catch missing video/keyframe/timeline/redaction pieces and see the next inspection path; `--require-ocr` and `--require-window-or-ax` tighten the audit when the live gate must prove OCR/window/AX observations. This helps authorized safe-window runs exercise recording-start guidance capture, redacted frame/video sidecar generation and post-write bundle reload, but it is not a replacement for real password, Secure Input or excluded-context product evidence.
