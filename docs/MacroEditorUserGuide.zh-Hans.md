# SparkleRecorder 宏编辑器操作说明

> 文档状态（2026-07-05）：当前宏编辑器用户说明仍保留。它描述单个宏编辑、OCR 选择和动作绑定；未来 AutomationEngine / FlowGraph / Resource Timeline 落地后，需要新增独立的编排用户流程说明。

## 基本心智模型

时间线里的每一行代表一个可理解、可排序、可编辑的动作，而不是底层事件噪音。

底层仍会保留 mouseDown、mouseUp、dragged、keyDown、scrollWheel 等 raw events，用于精确回放和诊断；编辑器默认展示的是语义动作，例如点击、拖拽、等待、点击文字、等待文本、验证文本。

## 插入位置

右侧插入动作默认插入到当前选中动作之后。

如果选中了多个动作，新动作会插入到选中范围的最后一个动作之后。没有选中任何动作时，新动作会追加到末尾。

## 常用动作

点击：点击固定坐标。适合窗口布局稳定、按钮位置明确的场景。

点击文字：查找目标文字，找到后点击匹配文字框的中心。适合按钮会移动、窗口会缩放、列表内容会变化的场景。

等待：纯等待一段时间，不发送鼠标或键盘输入。

等待文本：等待目标文本出现，然后继续执行后续动作。这个动作本身不会点击。

验证文本：立即检查目标文本是否满足条件。条件不满足时停止播放，并保存失败证据。

拖拽：保留按下、拖动轨迹、松开的完整手势。编辑起点/终点时会尽量保留原始轨迹形状。

按键：编辑一次键盘输入或快捷键。

## OCR 选择流程

点击文字拾取按钮后，系统会截图并识别目标窗口内容。

Picker 打开后会先倒计时 3 秒。倒计时结束前 OCR 框不会响应 hover 或点击，避免鼠标还停在 SparkleRecorder 自己按钮上时误触发。

倒计时结束后，把鼠标移动到目标文字上。橙色边框表示当前 hover 的文字框；淡橙色区域表示运行时优先搜索的区域。

点击文字后，动作会保存：

- 文字内容。
- 文字框位置。
- 搜索范围。
- 内容区归一化坐标。
- 坐标回退点。

## 找不到文字时

暂停：找不到目标文字时停止播放。这是默认策略，避免误点。

使用回退点：找不到文字时点击录入时保存的回退坐标。只建议在页面布局非常稳定时使用。

## 行为绑定

选中连续的一组动作后，可以使用“绑定为行为”把它们显示成一个行为块。

行为块适合表达一段完整意图，例如“打开菜单并选择导出”、“等待确认按钮并点击”、“拖动滑块后验证结果”。

绑定后排序会以整块为单位移动；“解除绑定”可以重新展开为独立动作。
