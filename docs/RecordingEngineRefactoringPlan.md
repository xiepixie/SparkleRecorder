# Recording Engine Refactoring Plan
## Implementation Plan v0.8
### From Direct Main Run-Loop Event Tap to Decoupled, Thread-Safe, and Semantic Event Recording

This document outlines the engineering architecture, low-level APIs, and target changes required to upgrade the event capture and recording engine in SparkleRecorder to support high-precision, low-latency, and thread-safe operations.

---

## 1. Core Architecture Upgrades

The upgraded recording engine transitions from a main-thread event tap loop to a multi-tiered, decoupled event pipeline:

```
Low-Level Input (OS)
       │
       ▼ (EventTapThread)
EventTapCallback (ListenOnly) ──> Filters out SparkleRecorder synthetic events
       │
       ▼ (RawInputEvent)
RawEventQueue (Thread-safe lock-free or serial queue)
       │
       ▼
RecorderCore Actor (Background Queue) ──> Throttled RecordingSurfaceTracker (Focused App metadata)
       │
       ▼ (Processes Gestures / Scroll Bursts / Key Unicode)
TimelineStore (CleanAction / RawEvents timeline)
       │
       ▼ (10 Hz throttle dispatch)
MainActor TimelineViewModel (SwiftUI UI presentation)
```

## 2. Key Refactoring Targets & Code Specifications

### A. Dedicated Event Tap Thread
- **Target File**: `Recorder.swift`
- **Issue**: The current event tap run-loop source is added to `CFRunLoopGetMain()`, which blocks or slows event capture during heavy UI rendering or window drags.
- **Fix**: Move the event tap run-loop source to a dedicated background thread (`EventTapThread`).
- **Target Code**:
  ```swift
  final class EventTapThread: Thread {
      private(set) var runLoop: CFRunLoop?
      private let semaphore = DispatchSemaphore(value: 0)
      
      override func main() {
          runLoop = CFRunLoopGetCurrent()
          semaphore.signal()
          CFRunLoopRun()
      }
      
      func waitAndGetRunLoop() -> CFRunLoop {
          semaphore.wait()
          return runLoop!
      }
  }
  ```
  When starting:
  ```swift
  let thread = EventTapThread()
  thread.start()
  let targetRunLoop = thread.waitAndGetRunLoop()
  CFRunLoopAddSource(targetRunLoop, runLoopSource, .commonModes)
  ```

### B. Prevention of Self-Recorded Synthetic Loops
- **Target Files**: `MouseKeyboardSynthesizer.swift` & `Recorder.swift`
- **Issue**: Synthetic events generated by SparkleRecorder can trigger the `CGEventTap` listener, causing feedback loops or falsely triggering user conflict detection.
- **Fix**: Tag all synthetic `CGEvent` instances with a unique magic header, and ignore them in the tap callback.
- **Target Code**:
  ```swift
  // In MouseKeyboardSynthesizer.swift
  let sparkleRecorderMagic: Int64 = 0x535041524B4C4521 // "SPARKLE!"
  let source = CGEventSource(stateID: .combinedSessionState)
  source?.userData = sparkleRecorderMagic
  
  // In Recorder.swift (event tap callback)
  let userData = event.getIntegerValueField(.eventSourceUserData)
  guard userData != 0x535041524B4C4521 else {
      return Unmanaged.passUnretained(event)
  }
  ```

### C. High-Precision Event Timing
- **Target File**: `Recorder.swift`
- **Issue**: Using system elapsed dates introduces timing jitter based on when the display timer flushes events.
- **Fix**: Read the absolute hardware timestamp of the `CGEvent` in nanoseconds since boot.
- **Target Code**:
  ```swift
  // Inside handle(type:event:)
  let eventTimestampNs = event.timestamp
  if startTimeNs == 0 {
      startTimeNs = eventTimestampNs
  }
  let elapsedSeconds = Double(eventTimestampNs - startTimeNs) / 1_000_000_000.0
  ```

### D. Multi-Mode Event Filtering & Trajectory Compression
- **Target File**: `EventGrouper.swift` & `Recorder.swift`
- **Proposed Modes**:
  1. `Click Mode`: Discards all `mouseMoved` events (only records clicks, scrolls, keystrokes).
  2. `Gesture Mode`: Records mouse drags and scrolls. Applies Douglas-Peucker path simplification to drag trajectories with a customizable distance threshold (e.g. `2.0` logical points) and time delta (e.g. `16ms`) to clean up path noise.
  3. `Raw Mode`: Captures full 60/120Hz coordinates.
- **Douglas-Peucker Simplification Implementation**:
  ```swift
  func simplifyPath(_ points: [CGPoint], epsilon: CGFloat) -> [CGPoint] {
      guard points.count > 2 else { return points }
      
      var maxDistance: CGFloat = 0
      var index = 0
      let end = points.count - 1
      
      for i in 1..<end {
          let distance = perpendicularDistance(from: points[i], lineStart: points[0], lineEnd: points[end])
          if distance > maxDistance {
              index = i
              maxDistance = distance
          }
      }
      
      if maxDistance > epsilon {
          let left = simplifyPath(Array(points[0...index]), epsilon: epsilon)
          let right = simplifyPath(Array(points[index...end]), epsilon: epsilon)
          return left.dropLast() + right
      } else {
          return [points[0], points[end]]
      }
  }
  ```

### E. Detailed Scroll Wheel Representation
- **Target File**: `RecordedEvent.swift`
- **Issue**: Multiplying line delta by 12 loses precision for high-resolution scroll wheels and trackpad scroll actions.
- **Fix**: Store full scroll metadata payload.
- **Target Code**:
  ```swift
  public struct ScrollPayload: Codable, Equatable {
      public var pointDeltaX: Int32
      public var pointDeltaY: Int32
      public var lineDeltaX: Int32
      public var lineDeltaY: Int32
      public var phase: Int32?
      public var momentumPhase: Int32?
  }
  ```

### F. Unicode Keystroke Mappings
- **Target File**: `RecordedEvent.swift`
- **Issue**: Physical keyCodes map to different characters on non-US keyboard layouts.
- **Fix**: Save the unicode scalar string alongside the raw keycode.
- **Target Code**:
  ```swift
  // In Recorder.swift (event tap callback)
  var unicodeString: String? = nil
  var actualLength = 0
  let maxLen = 4
  var unicodeBuffer = [UniChar](repeating: 0, count: maxLen)
  event.keyboardGetUnicodeString(maxStringLength: maxLen, actualStringLength: &actualLength, unicodeString: &unicodeBuffer)
  if actualLength > 0 {
      unicodeString = String(utf16CodeUnits: unicodeBuffer, count: actualLength)
  }
  ```

### G. Safe Event Tap Teardown
- **Target File**: `Recorder.swift`
- **Issue**: Stopping recording without invalidating `CFMachPort` can leak memory or leave duplicate event sources attached.
- **Fix**: Implement a strict cleanup sequence.
- **Target Code**:
  ```swift
  func stopRecording() {
      guard isRecording else { return }
      if let tap = tap {
          CGEvent.tapEnable(tap: tap, enable: false)
      }
      if let source = source, let runLoop = tapThread?.runLoop {
          CFRunLoopRemoveSource(runLoop, source, .commonModes)
      }
      if let tap = tap {
          CFMachPortInvalidate(tap)
      }
      
      self.tap = nil
      self.source = nil
      self.isRecording = false
      
      // Stop surface tracker & timers...
  }
  ```

## 3. Preflight System Permissions Checks

The user interface will execute checks for each category of permissions before entering recording/playback states:

```swift
// Listen events preflight check
func checkListenEventPermission() -> Bool {
    if #available(macOS 14.0, *) {
        return CGPreflightListenEventAccess()
    }
    return AXIsProcessTrusted()
}

// Request listen permission
func requestListenEventPermission() {
    if #available(macOS 14.0, *) {
        CGRequestListenEventAccess()
    } else {
        let options = [kAXTrustedCheckOptionPrompt.takeUnretainedValue() as String: true] as CFDictionary
        AXIsProcessTrustedWithOptions(options)
    }
}
```

## 4. Modified Implementation Phases

We partition our implementation steps into incremental milestones to keep the codebase testable and prevent regression bugs:

### Phase 1: Coordinate Space & Core Event Tap Upgrades
- Implement the `EventTapThread` and move tap execution off the main run-loop.
- Implement the `eventSourceUserData` loopback filter using the magic header.
- Upgrade `RecordedEvent` data schema to store `windowLocalX/Y` and `windowNormalizedX/Y` (flat variables).
- Implement `RecordingSurfaceTracker` with a throttled `150ms` loop.
- Rewrite `PointResolver` to return `Result<CGPoint, PointResolveError>` and fail safely if coordinates map out of bounds.

### Phase 2: Multi-App Surfaces Lookup
- Integrate the `surfaces` map in `SavedMacro`.
- Map events to `surfaceId` and handle focus transitions.
- Lock `surfaceId` on `mouseDown` during dragging gestures.

### Phase 3: Scroll & Keyboard Representation upgrades
- Add `ScrollPayload` and unicode mapping attributes to `RecordedEvent`.
- Replace basic `* 12` scroll scaling with structured payload playback.

### Phase 4: Runner & ScreenCaptureKit Integration
- Implement lazy window resolution/activation inside `PlaybackContext`.
- Integrate `SCScreenshotManager` for single-frame verification and failure snapshots.
